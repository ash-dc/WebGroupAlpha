ESRI's Experience Builder

Experience builder was found to be the most up to date and appropriate enviroment to create a simple web applicationtion that would allow public users to view their properties and the associated flood risk. 2 options were created initially to explore functionality and determine what the final solution would look like.

Option 1

Using the projected data layers collected by the group, the buildings were clipped within the 50m flood plain. All analysis and symbology was completed in ArcGIS Pro. Once the ArcGIS Pro document was published to ArcGIS Online, data was added to a new Experience. A template titled Jewelry Box was used to create this app. Several widgets were used to visualize the data, including a map widget to view data layers, a list widget to view and zoom to the watersheds within the study area, and the map layers widget allowing the user to turn layers on and off within the map. Additionally, a feature info widget was used to view details of and zoom to buildings within the 50m flood plain. Text and pictures were used to enhance and customize the appearance of the app for Conservation Halton.

Option 2

This prototype used the same raw data from the group.The building count in every watershed was formulated using the Summarize Within Tool. The symbology and analysis was again, completed in ArcGIS Pro. The ArcGIS Pro document was published to ArcGIS Online and a new Experience was created. A Blank template was used and a map widget was used to visualize the data.A Feature Info Widget was used to view information(building count) about each watershed. The Widget was also configured to zoom to the watershed on it's selection. The Legend widget was fit into the display. A Share widget was placed into the map section for additional functionality. A filter widget was added to filter certain watersheds.

ESRI's Operations Dashboard

To create an internal solution that could display numerous statistics and calculations, Operations Dashboard was determined to be the best platform. 3 preliminary dashboard options were created to explore functionality and help to form the final solution.

Option 1

This operations dashboard focused on utilizing mock data regarding building types and their associated risk to attempt to compute a more complex vulnerability calculation.
An indicator widget was used to count the number of affected buildings (overlapping the 50m flood plain data) within the viewshed of the map. A pie chart widget summarized the classification of each of the affected buildings. Finally, a gauge widget was used to attempt to calculate a simple vulnerability indicator. The gauge sums the risk scores (based on building type, ranging from 1 to 3 with 3 being the most vulnerable) of buildings within the flood plain and divides by the total number of buildings in the area.
However, there were errors with the data sets being used that prevented this widget from calculating accurately. Other limitations of this application are an inability to view these statistics by watershed or municipality.

Option 2

This operations dashboard uses pre-processed data from a web map created in AGOL. The summarize within tool was used to find areas in square meters of building polygons that are within the flood buffer zones for 3 buffer distances: 10m, 30m, 50m. The Web Map displays the areas of risk associated with the total area of house polygons (in Sq. Meters) within a stretch of the river segment. This allows users to easily see what areas have a higher risk for a 10m, 30m, or 50m flood buffer around the river. Sub watersheds where symbolized to show each of the different watershed zones. This is for environmentalists or the conservation authority to see what watersheds have more risk of flood damage to buildings. When developing the Pie chart, I configured the category field to display the 7 largest Watersheds, opposed to all for simplicity and clarity. I also added functionality to the pie chart so users can click a pie segment and it will zoom to the watershed selected, flash, and display it on the map. It also shows a popup of information related to that area of watershed. Unclicking the pie segment brings the map back to the default extent.
